Projects

I’ve focused mainly on building practical, lightweight tools that solve specific problems for me, even if over time they’ve fallen out of maintenance and no longer function as intended. I tend to approach development from a utilitarian mindset: if I encounter friction in my workflow, I would rather write a small tool to remove that friction than rely on heavier, more generalized software. I created Newsfy as a C++ terminal application that lets me quickly check Arch Linux news directly from the command line without needing to open a browser, because I prefer staying in a minimal, keyboard-driven environment where everything is fast, scriptable, and distraction-free. It was as much about reinforcing my comfort with C++ and terminal-based interfaces as it was about the convenience of having updates accessible in a streamlined format. With WDIJD, I built a Windows-based logging utility in C++ that tracks commands and application usage, primarily to better understand and document system activity on my own machine. That project reflected my interest in observing patterns in how I use my system, as well as my curiosity about lower-level interaction with the operating system. XMRT is my cryptocurrency tracker centered around Monero, designed to monitor relevant data in a straightforward and efficient way without relying on bloated web dashboards. I aimed to keep it simple and performance-focused, surfacing only the information that actually mattered to me. Across these projects, I consistently prioritize simplicity, performance, and direct system interaction over building large, complex applications with extensive abstractions. I’m more interested in tools that feel personal and purposeful than in creating polished, enterprise-scale software, and each repository reflects a hands-on approach to learning by building something immediately useful to my own workflow. If you wish to explore them further, you can find them on my github.
